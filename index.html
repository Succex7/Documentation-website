Introduction to HTML

HTML stands for HyperText Markup Language. It is the standard language used to create and structure content on the web. Every website you've ever visited is built on HTML, whether simple or complex.
HTML works by using "elements" that tell the browser what to show: text, messages, buttons, links, forms, etc.. It is the skeleton of every web page.

Basic Structure

Every HTML document has a simple structural pattern that the browser understands. The structure includes a document type declaration, a root element, a head section, an a body section.
The head contains information about the webpage. The body contains everything you want users to see: text, images, links, etc..
Example: 

                
                    <!DOCTYPE html>
                    <html lang="en">
                    <head>
                        <meta charset="UTF-8">
                        <meta name="viewport" content="width=device-width, initial-scale=1.0">
                        <title>Document</title>
                    </head>
                    <body>
                        
                    </body>
                    </html>

HTML Elements

Elements are the building blocks of every webpage.
An element is made up of:

  • an opening tag
  • some content
  • and a closing tag
Some elements wrap around content, while some stand alone.
Elements tell the browser what each part of the webpage represents, either a paragraph, a heading, an image, and so on.
Example: 
                    
                        <header></header>
                        <nav></nav>
                        <main></main>
                        <section></section>
                        <article></article>
                        <div></div>
                        <p></p>

HTML Attributes

Attributes are extra instructions you attach to an element.
They provide additional information or modify how elements behaves.
Examples of what elements can do:

  • Give an element an ID
  • Add a class for styling
  • Add a link to an image
  • control how a link opens
Attributes always go inside the opening tag, an they come in the name/value pairs.
Example: 
                    
                        <p class="p1">The attribute here is class</p>

HTML Headings and Paragraphs

headings help structure your page and show th importance of content.
They run from the biggest (h1) to the smallest (h6).
Paragraphs are used for writing blocks of text.
They help separate ideas, improve readability, and keep your content organized.
Example: 

                    
                        <h1>This is h1 heading</h1>
                        <h2>This is h2 heading</h2>
                        <h3>This is h3 heading</h3>
                        <h4>This is h4 heading</h4>
                        <h5>This is h5 heading</h5>
                        <h6>This is h6 heading</h6>

HTML Links

Links allow users to move from one page to another, or to different sections of the same page.
They can point to websites, phone numbers, email addresses, files, or internal locations on your page.
Links are one of the core features that make the web interconnected.
Examples: 

                    
                        <a href="#">link</a>

HTML Images

Images add visual meaning t your webpage. You can display photos, illustrations, icons, and any media file supported by the browser.
Every image needs a source and alternate text. The alternate text is especially helpful for accessibility and for situations where the image fails to load.
Example: 

                    
                        <img src="image.jpg" alt="My picture">

HTML List

Lists help to organize information in a clean and readable way.
There are two main types:

  • Ordered lists (for numbered items)
  • Unordered lists (for bullet points)
They are ideal for steps, summaries, menus, and grouped information.
Example: 
                    
                        <ol>
                            <li>First ordered list item</li>
                            <li>Second ordered list item</li>
                            <li>Third ordered list item</li>
                        </ol>

                        <ul>
                            <li>First unordered list item</li>
                            <li>Second unordered list item</li>
                            <li>Third unordered list item</li>
                        </ul>

HTML Tables

Tables allow you to present data in rows columns. They're great for schedules, price lists, comparison charts, and structured information.
Tables are made of rows, header cells, and data cells.
Example: 

                    
                        <table>
                            <tr>
                                <th>Heading 1</th>
                                <th>Heading 2</th>
                            </tr>

                            <tr>
                                <td>Data 1</td>
                                <td>Data 2</td>
                            </tr>
                        </table>

HTML Forms

Forms are how users interact with your website. They let people enter data like names, emails, passwords, messages, or search items.
Forms contain different input types such as text fields, checkboxes, radio buttons and submit buttons.
They are essential for login systems, contact pages, dashboards, and anything that collects information.
Example: 

                    
                        <form>
                            <label for="name">Name</label>
                            <input type="text" id="name" placeholder="Enter your name" required/>

                            <label for="email">Email</label>
                            <input type="email" id="email" placeholder="Enter your email" required/>

                            <label><input type="radio" name="recommend">This is a radio button 1</label>
                            <label><input type="radio" name="recommend">This is a radio button 2</label>
                        </form>

HTML Semantic Tags

Semantic tags help describe the meaning of content. They make webpages more readable for both humans and search engines.
Examples include:

  • chunks of content
  • navigation menus
  • headers
  • footers
  • highlighted sections
Semantic tags improve structure, SEO and accessibility.
Examples: 
                    
                        <header>header</header>
                        <nav>navigation</nav>
                        <main>main content</main>
                        <section></section>
                        <article></article>

HTML Comments

Comments allow you to leave notes inside your code, these notes are ignored by thr browser and only visible to developers.
They're useful for reminders, explanations, disabling code temporarily, and organizing large projects.
Example: 

                    
                        <!-- This is a comment -->

HTML Audio and Video

Modern HTML lets you embed audio and video without extra plugins.
You can play songs, podcasts, tutorials or show movies inline on your webpage.
You can also enable controls like play, pause or volume.
Example: 

                    
                        <audio controls>
                            <source src="myaudio.mp3" type="audio/mp3">
                            <source src="myaudio.ogg" type="audio/ogg">
                        </audio>
                        <video controls poster="thumbnail.jpg">
                            <source src="myvideo.mp4" type="video/mp4">
                            <track src="subtitles_eng" kind="subtitles" srclang="en" label="english"></track>
                        </video>

HTML pre and code Tags

These two tags allows you to display code exactly as it is typed.

  • pre tag keeps spacing and line breaks
  • code tag identifies the content as code
Together, they are perfect for documentation websites, tutorials and examples. 
                    

                        <pre>
                            <code>
                                <p>This is a code</p>
                            </code>
                        </pre>

HTML Best Practices

Some good habits to follow as you grow:

  • Use proper indentation
  • Write clean, readable codes
  • Add comments for clarity
  • Use semantic tags for good structure
  • Always include alt text for images
  • Validate your HTML regularly
These help you build professional, accessible, developer-friendly websites.
